Detailed record of how this collection (and eventually the lecture) was built — itself raw material for the meta-lesson (see TODO.md).

## 2026-06-11 — Session 1

- **Global survey.** Inventoried `~/.claude/`: 6 CLAUDE.md rule sections, 5 custom skills, settings.json, custom statusline script, codex-reviewer MCP. No global hooks/agents/commands/keybindings existed.
- **Repo sweep.** A background sonnet Explore agent (chosen per the just-established model-selection rule) cataloged ~30 repos under `$GH` at depth ≤ 2: CLAUDE.md tiers, `.claude/` contents, project memories, `AI/` directories. Raw output: `notes/per-project-customizations-sweep.md`.
- **Structure decided.** Two artifacts per item: `lessons/NN-name/LESSON.md` (story/reasoning) + a directly usable artifact file. Lessons end with a first-person "From the other side of the prompt" comment from Claude (user's request, after liking an agent-perspective aside in discussion).
- **Corrections collected.** (a) "Interactive" = one-at-a-time discussion, never AskUserQuestion batches — notably the same correction already existed in ccif-patent's project memory, demonstrating that per-project memory doesn't propagate. (b) Resist verbosity in durable docs; watch CLAUDE.md/MEMORY.md sizes. Both saved to project memory.
- **Findings fed back into the real config.** The `$GH` topology (group folders, max depth) was undocumented → global CLAUDE.md updated. Paths that looked stale (`~/repositories/...`) turned out to be the Linux-account convention → Lesson 2, plus a TODO to set `$GH` on Linux.
- **Lessons written.** 01 (repo location convention), 02 (one name, many machines).
- **Lesson 03 written** (never-silently-skip). Origin story collected in discussion: the `\pacs{}` incident ("I don't know what this is — cover it up"), and the repeated point-it-out → "yes, that contradicts my clear instruction" → no-change loop that led to the diagnosis (instruction loses to the don't-crash objective) and the dedicated-audit-skill solution. Skill shipped verbatim as payload (`no-silent-skip-review/SKILL.md`); first lesson whose install prompt carries a payload file.
- **Lesson 04 written** (stop-and-think-before-building). User remembered the LLM-wrapper incident (built a provider-abstraction class; an existing package already did it) but not the N>50 origin; a grep of project memories recovered it — astrothesaurus-dev's `feedback_batch_api.md`: 2,312 × `claude -p` calls at ~13K overhead tokens each, batch API never suggested. The global rule turned out to be a near-verbatim promotion of that memory's "How to apply" — first documented instance of the correction → project memory → global rule pipeline (OBSERVATIONS #7).
- **Lesson 05 written** (standing vetoes, generalized from the no-Homebrew rule). Origin: accumulated friction — agents constantly proposed `brew install`, the answer was always no. Lesson framed as Lesson 1's negative-space twin (declared refusals vs declared facts); key structural points: ban the class not the instance, and always state the alternative route.
- **Lessons 06–08 written** (review policy, split three ways: loop, diversity, empty context). Origin story collected in one rich discussion turn: the manual codex copy-paste loop and the "I'm a human IPC" realization → automation via subagents + MCP; the opus-is-best assumption broken by sonnet finding what opus and codex both missed ("not better/worse — different; table stakes"); the nit-pick pressure vs stops-short tension behind the dry-round rule and "major issues" framing; and the over-stuffed review prompt discovered by asking to see it — leading to the two-word prompt and the explicit "DO NOT add more" fight with the delegating agent. Four new observations (#9–12), including the lecture's emerging through-line: robust practices decline to negotiate with the agent's good intentions.
- **Lesson 09 written** (findings are claims — provenance labeling). Origin: during the manual era, pasted codex reviews arrived with the user's authority and were implemented as commands; the fix is labeling ("from an AI reviewer, not me — verify"), which licenses pushback grounded in the working agent's context. Completes the review-policy arc (06–09); the 08/09 symmetry (clean sight vs institutional memory) became OBSERVATIONS #14. Correction recorded: ccif-patent's hallucinated-citations memory is recent, unrelated to this rule's origin — an agent guess that pattern-matched and was wrong.
- **Lesson 10 written** (encode your convictions; enum rule as worked example). Origin: pre-AI conviction, not an agent incident — a third species of lesson (1: environment facts, 2: failure-mode defenses, 3: encoded taste). Key user reasoning captured: integer PKs can't be guaranteed stable so meaning gets lost; a string constant at the boundary is human-readable and repairable — enum in code, `.value` at boundaries. Rule anatomy distilled: why + exceptions + application pressure.
- **Lesson 11 written** (session lifecycle: /endsession + /park, both skills shipped verbatim as payloads after a clean personal-info grep). Origin: many concurrent projects/sessions (incl. manually-arranged parent/child sessions); windows stayed open because final summaries needed unavailable attention → RAM pressure on a 16GB laptop; reboots meant tying up a dozen+ loose ends at once. endsession grew from the habitual "anything left to save before closing?"; park from the session-as-document metaphor — save the work summary + the verbatim undealt-with final summary, not the session itself. Also: Lesson 8 upgraded — the "higher level suggestions" line promoted from footnote to gold-standard feature (correctness review AND "is this the right approach?", pays off almost every time).
- **Correction:** `AI/in_flight/` is the implementation detail of `/park` (HOW parking works), not part of a separate `AI/` directory convention. The upcoming AI-convention lesson covers `START_HERE.md` and its siblings; in_flight belongs to Lesson 11.
- **Session 1 parked** at the user's prompt ("worth picking up in a clean session?") — itself a live demonstration of Lesson 11.
- **Artifact format redesigned.** The user spotted that the first artifact conflated human instruction with agent-facing payload. Resolved: artifacts are now agent-directed `install-prompt.md` files — the recipient pastes one into their own Claude session, and the agent inspects the system, interviews (one question at a time), adapts to differences, shows the text, and merges into config. Self-localizing: no hand-edited placeholders, and the deliverable is consumed through the very tool the lecture teaches. LESSON.md remains the human-facing document.

- **Lesson 13 written** (token economy / trace reading). Origin: subscription rate limits hit constantly — tokens are working time, not money; the user reads live traces hunting "blind groping." Observations #22–23 added. Rule deployed live to the global CLAUDE.md (third live deployment, after $GH topology and the clone test).
- **Lesson 14 written** (model economics / budget routing). Tier map: constrained tier for judgment, sonnet workhorse (pinned in config, e.g. sql-tesseretica), haiku trivial, err-upward tiebreak. Third budget: API credits for anything batchable with no human waiting — Batch API 50% + prompt front-loading (static prefix, custom tail → cache reads ≈0.1× input price). **Fact-check via the claude-api skill** before shipping numbers: batch 50% confirmed; cache read ≈0.1× confirmed; user's "~24K minimum" corrected — Anthropic minimums are 1,024–4,096 tokens by model; the ~24K figure likely migrated from Gemini's ~32K context-caching minimum (user runs Gemini evals via OpenRouter). - **Lesson 15 written** (tool kit / stop-don't-degrade). Origin: the user watched agents reach for the best tool (rg, jq, mediainfo…), find it missing, and *degrade* through ever-worse methods instead of stopping — he'd interrupt, install the tool, continue, and wished the agent would stop and ask for the install itself.
